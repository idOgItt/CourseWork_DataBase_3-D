
### План разработки проекта

#### Этап 1: Анализ и проектирование

1. **Сбор требований:**
    - Определить роли пользователей:
        - Администратор: управление пользователями, модерация отзывов, создание резервных копий и управление скидками.
        - Клиент: регистрация, просмотр каталога, добавление моделей в корзину, оформление заказов, оставление отзывов.
        - Гость: просмотр каталога.
    - Уточнить функционал:
        - Каталог моделей с возможностью поиска, фильтрации и сортировки.
        - Управление корзиной, оформление заказов, отзывы и рейтинги.
    - Задокументировать требования в формате спецификаций.

2. **Проектирование базы данных:**
    - Определить сущности (модели, категории, пользователи, заказы, отзывы, платежи) и их связи.
    - Создать ER-диаграмму.
    - Подготовить SQL-скрипты для создания таблиц, представлений, функций и триггеров.
    - Обеспечить соответствие 3NF и выше для всех таблиц.

3. **Архитектура приложения:**
    - Разделить на три уровня: база данных, серверное приложение, клиентское приложение.
    - Выбрать технологии: Spring Boot для backend, JavaFX для frontend.
    - Определить точки интеграции через gRPC API.

---

#### Этап 2: Настройка инфраструктуры

1. **Инициализация проекта:**
    - Создать структуру проекта в соответствии с указанной схемой.
    - Настроить Git-репозиторий.

2. **Подготовка окружения:**
    - Написать Dockerfile для backend, frontend и базы данных.
    - Настроить `docker-compose` для разработки и продакшена.

3. **Инициализация базы данных:**
    - Написать `init.sql` для создания таблиц.
    - Добавить тестовые данные в `data.sql`.
    - Настроить миграции с использованием Flyway.

4. **CI/CD:**
    - Настроить автоматическое тестирование и сборку с использованием GitHub Actions.

---

#### Этап 3: Разработка серверной части (backend)

1. **Настройка Spring Boot:**
    - Инициализировать базовый проект Spring Boot.
    - Настроить подключение к PostgreSQL.
    - Подключить Hibernate и создать модели данных.

2. **Создание слоев приложения:**
    - **Controller:** Rest API для взаимодействия с клиентами.
    - **Service:** Бизнес-логика.
    - **Repository:** Доступ к данным через Hibernate.

3. **Реализация API:**
    - CRUD-операции для всех основных сущностей.
    - Реализация аутентификации и авторизации.
    - Добавление глобальной обработки ошибок.

4. **Тестирование:**
    - Написать юнит- и интеграционные тесты.
    - Тестирование с помощью postman
    - Тестирование API
    - Тестирование БД и миграций

---

#### Этап 4: Разработка клиентской части (frontend)

1. **Создание проекта:**
    - Инициализировать JavaFX проект.
    - Инфраструкта
    - Создать шаблоны страниц: каталог, вход, корзина, админ-панель.

2. **Реализация интерфейса:**
    - Настроить взаимодействие с API.
    - Добавить функционал поиска, фильтрации, работы с корзиной и оформления заказов.
    - Реализовать управление состоянием.

3. **Тестирование:**
    - Написать модульные и e2e тесты.

---

#### Этап 5: Дополнительный функционал

1. **Резервное копирование и восстановление базы:**
    - Реализовать хранимые процедуры для создания и восстановления резервных копий.
    - Добавить интерфейс для этой функции в админ-панель.

2. **Безопасность:**
    - Шифрование данных (пароли, платежи).
    - Защита от SQL-инъекций и XSS-атак.

3. **Оптимизация производительности:**
    - Настроить кэширование.
    - Оптимизировать SQL-запросы.

---

#### Этап 6: Завершение проекта

1. **Тестирование и исправление ошибок:**
    - Провести нагрузочные тесты.
    - Исправить выявленные баги.

2. **Документация:**
    - Написать `user_guide.md` для пользователей.
    - Создать `developer_guide.md` для разработчиков.
    - Подготовить `schema_description.md` для базы данных.

3. **Подготовка к демонстрации:**
    - Разработать сценарии демонстрации (регистрация, заказ, восстановление базы).
    - Подготовить презентацию.

---

#### Этап 7: Деплой

1. **Настройка продакшена:**
    - Настроить `compose.prod.yml`.
    - Развернуть проект на сервере.

2. **Финальные шаги:**
    - Провести финальное тестирование в продакшене.
    - Подготовить отчет по проекту.
